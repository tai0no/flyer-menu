# AGENTS.md（Repository Guidelines）

## 言語
- 返答・説明・出力は原則日本語。
- 不明点があれば推測せず、前提/仮定を明記する。

## 進め方
- まず作業計画（箇条書き）→小さく実装→自己レビュー（/review）→次の変更、の順で進める。
- 変更は最小単位に分け、意図と影響範囲を短く説明する。
- コマンド実行・ファイル編集は、必ず「目的」「実行内容」を先に示し、承認を待つ。

## コード提示ルール（重要）
- コードを提示する場合は **全文** を示す。
- どこに変更を入れるか（ファイル名・変更点）を明示する。

## コーディング規約
- TypeScript優先。命名は可読性重視（変数/関数=camelCase、コンポーネント=PascalCase）。
- フォーマット/静的解析（例：Prettier/ESLint）が導入されている場合はそれに従う。

## テスト/品質
- 変更後に lint / typecheck / test が通る状態を維持する。
- 例外処理と入力バリデーションを重視する。

## セキュリティ
- 秘密情報（APIキー等）をコミットしない。必要なら .env.local と .env.example を使う。

## このプロジェクト固有（MVP方針）
- 対象：ライフ/食品館あおば/イトーヨーカドーの店舗ページからチラシ情報を取得する。
- サイト構造が異なるため、取得ロジックは providerKey（識別子）で分岐する「アダプタ方式」で実装する。
- 取得できたチラシ（PDF/画像）は後段でAI（Gemini）に渡す前提で扱う。
